# RFCs — Architecture decision records for OpenConstructionERP v1.9+

Each RFC captures the **rationale** behind a non-trivial design decision. Format:

```
# NN — Title

**Status:** draft | approved | implemented | superseded
**Related items:** ROADMAP_v1.9.md #NN
**Date:** YYYY-MM-DD

## Context
What's the current state? What's broken / missing?

## Options considered
1. Option A — pros / cons
2. Option B — pros / cons
3. Option C — pros / cons

## Decision
Chosen option + 1-paragraph rationale.

## Implementation sketch
File-level plan: what modules change, what new models/schemas/endpoints.

## Testing plan
Unit + E2E + visual + a11y coverage plan.

## Risks / follow-ups
```
